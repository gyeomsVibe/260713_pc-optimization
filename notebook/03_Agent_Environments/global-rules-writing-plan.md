# 글로벌 룰 작성 계획서

> 대상: 노트북의 Antigravity · Codex · Claude Code
> 상태: **영문 정본 v2.0.1 작성·장착·검증 완료 — 2026-07-15**
> 기준 문서: 각 에이전트의 환경 기록(2026-07-15 확인)

## 1. 목표와 범위

세 에이전트가 같은 안전·품질 기준으로 작업하되, 각 도구의 설정 방식과 권한
경계를 침범하지 않도록 글로벌 룰을 설계한다. 이 문서는 구현 순서와 검증 기준을
정한 계획서이며, 플러그인·MCP·권한 설정을 실제로 바꾸는 지침은 아니다.

## 2. 비교 결과

| 구분 | Antigravity | Codex | Claude Code |
|---|---|---|---|
| 주 작업 방식 | IDE 통합형 에이전트 | Desktop/CLI 에이전트 | CLI 에이전트 |
| 로컬 관리 영역 | `.agents/`, IDE 확장 | `~/.codex/` | `.claude/` |
| 외부 도구 연결 | `.agents/mcp_config.json` | 플러그인·MCP·Desktop 커넥터 | 플러그인·MCP·claude.ai 커넥터 |
| 주 위험 | IDE 확장·죽은 MCP 참조 | 플러그인 과다·인증 포함 설정 | 과도한 자동 실행 권한·계정 커넥터 |
| 특화 안전장치 | 브라우저 JavaScript 실행은 항상 확인 | 샌드박스·승인 정책 유지 | 허용 명령어 목록을 최소화 |

### 확인된 문서 정합성 과제

- Codex 환경 문서의 실제 플러그인 캐시 위치와 CLI 명령은 구현 전에 다시 확인한다.
  문서에 적힌 `disable`/`enable`은 현재 CLI의 명령 체계와 다를 수 있으므로, 검증 없는
  명령어를 글로벌 룰에 넣지 않는다.
- Antigravity의 `.agents/`에는 머신 고유 값이 포함될 수 있으므로, 구조만 문서화하고
  원문은 커밋하지 않는다.
- Claude Code의 계정 커넥터는 로컬 파일과 별개이므로, 로컬 글로벌 룰이 원격 계정
  설정을 임의로 변경하지 않도록 분리한다.

## 3. 공통 글로벌 룰 초안 원칙

1. **읽기 우선**: 관련 파일·현재 설정·기존 변경을 확인한 뒤에만 수정한다.
2. **도메인 격리**: 각 에이전트는 자기 관리 영역 밖의 설정을 수정하지 않는다.
3. **최소 권한**: 필요한 도구만 활성화하고, 새 플러그인·패키지·MCP는 목적과 영향을
   설명한 뒤 추가한다.
4. **비밀 보호**: 토큰, API 키, 인증 파일, 개인 식별 정보는 읽기·출력·커밋하지 않는다.
5. **안전한 변경**: 삭제·배포·권한 변경·외부 쓰기 작업은 영향·복구 가능성·승인을
   확인한다.
6. **검증과 기록**: 변경마다 최소 검증을 실행하고, 결과·롤백 방법·미검증 항목을
   문서에 남긴다.
7. **작업 분리**: 노트북과 데스크톱의 설정·플러그인·문서는 공유하지 않고 각 환경에서
   독립적으로 판단한다.

## 4. 도구별 글로벌 룰 설계 방향

### Antigravity

- `.agents/`의 MCP 등록은 실제 IDE 확장과 경로가 존재할 때만 유지한다.
- 브라우저 JavaScript 실행은 현재의 확인 우선 정책을 보존한다.
- IDE 확장은 언어 서버·AI 도구의 중복 여부와 유휴 리소스 사용량을 기준으로 정리한다.

### Codex

- `~/.codex/`의 인증·상태 파일은 문서화하거나 저장소에 추가하지 않는다.
- 플러그인 변경 전 `codex plugin --help`와 현재 목록으로 지원 명령과 영향 범위를
  확인한다.
- Desktop 커넥터, 로컬 플러그인, 공유 마켓플레이스 캐시를 구분해 불필요한 캐시 삭제로
  정상 플러그인 상태를 손상시키지 않는다.

### Claude Code

- `.claude/settings.local.json`의 자동 실행 허용 목록은 최소 범위로 유지하고, 새 명령은
  반복 사용·안전성·롤백 필요성을 기준으로 추가한다.
- 로컬 플러그인, MCP, claude.ai 계정 커넥터를 서로 다른 관리 대상으로 취급한다.
- 계정 커넥터 변경은 로컬 정리 작업과 분리하고, 실제 사용 여부를 먼저 확인한다.

## 5. 실제 작성·적용 순서

1. **기준선 기록**: 각 도구의 버전, 활성 플러그인, MCP 목록, 적용 중인 지침 파일을
   읽기 전용으로 기록한다.
2. **공통 규칙 확정**: 3장의 공통 원칙을 짧은 필수 규칙으로 확정한다.
3. **도구별 규칙 변환**: 같은 원칙을 Codex용 `AGENTS.md`, Claude Code용 지침 파일,
   Antigravity가 지원하는 프로젝트 규칙 위치에 각각 맞춰 번역한다. 위치는 실제 지원
   여부를 확인한 뒤 확정한다.
4. **드라이런 검토**: 파일 변경 없이 각 규칙이 플러그인 설치, MCP 추가, 삭제, 배포,
   비밀 보호 상황에서 어떻게 동작하는지 사례로 점검한다.
5. **최소 적용**: 도구별 규칙 파일만 작은 단위로 적용하고, 설정·플러그인 변경은 별도
   작업으로 분리한다.
6. **검증·롤백**: 새 세션에서 지침이 로드되는지 확인하고, 문제가 있으면 직전 규칙
   파일만 되돌린다.

## 6. 완료 기준

- 세 에이전트가 공통 안전 원칙을 공유하지만 서로의 설정 파일을 수정하지 않는다.
- 각 규칙 파일은 해당 도구가 실제로 지원하는 위치·명령어만 사용한다.
- 인증 정보와 머신 고유 값이 Git 변경 목록에 포함되지 않는다.
- 플러그인·MCP·권한 변경은 별도 승인·검증·롤백 단위를 가진다.

## 7. 적용 후 확인 항목

- 각 도구가 프로젝트별 지침 파일을 자동으로 읽는 정확한 경로와 우선순위
- Codex 플러그인 관리 명령 및 캐시 경로의 현재 버전 기준
- Claude Code 허용 명령어 목록의 실제 사용 빈도와 위험도
- Antigravity의 프로젝트 규칙 파일 지원 여부 및 IDE 확장 상태
