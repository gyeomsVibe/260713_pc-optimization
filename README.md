# PC 환경 최적화 통합 작업대 (PC Environment Optimization Workspace)

이 저장소는 노트북(Notebook) 및 데스크톱(Desktop) PC 환경에서 발열 제어, 지연 감소(Latency Reduction), 성능 및 개발 생산성 극대화를 달성하기 위한 환경 최적화 및 튜닝 기록을 통합 관리하는 공간입니다.

각 환경의 대상 하드웨어 사양과 튜닝 스크립트, 그리고 복구(Rollback) 경로는 개별 작업대 내에서 안전하게 격리되어 관리됩니다.

## 🛠️ 공통 최적화 원칙
- **비침범성**: 기존의 유효한 하드웨어 레벨 언더볼팅/성능 세팅은 최대한 보존하며 소프트웨어 스택과 전원 관리 스킴을 먼저 조율합니다.
- **안전성 (Safe-first)**: 모든 최적화 적용 전에 반드시 백업(Rollback) 경로를 확보하고 복구 가능 여부를 검증합니다.
- **독립성**: 환경별 스크립트와 설정 파일은 각 하위 폴더(`notebook/scratch/`, `desktop/scratch/`) 내에서 동작하여 간섭을 방지합니다.
- **에이전트 도메인 격리**: 여러 AI 에이전트를 함께 쓸 때, 한 에이전트는 다른 에이전트의 설정(`.claude/`·`~/.codex/`·`.agents/`)을 건드리지 않으며 각자 전용 문서에서만 환경을 관리합니다.

> 머신 고유값(호스트명·절대경로·사용자명 등)을 담은 설정 파일은 git에서 제외하고 **구조만 문서화**합니다. 자세한 위생 규칙은 [`.gitignore`](.gitignore) 및 각 에이전트 환경 문서를 참조하세요.

## 🚀 최근 최적화 적용 및 트러블슈팅 히스토리 (2026-07-16)

바탕화면 우클릭 메뉴 최적화 및 윈도우 탐색기 폴더 정렬 최적화 작업을 진행하며 도출된 **실패 사례(Failure Cases)와 최종 해결책(Solutions)**을 기록합니다.

### 🚫 실패 사례 & 해결책 요약

#### 1. 새로 만들기(New) 메뉴 복구 누락 및 오작동
- **증상/원인**: 새로 만들기 메뉴의 식별자(CLSID)를 바로가기용 식별자인 `{D67D100C-CC88-11D0-BE25-00C04FC8F20C}`로 혼동해 오기입했고, 파워쉘의 이스케이프 이중 처리 오류로 인해 HKLM 쓰기 배치가 실행되지 않아 변경이 유실됨.
- **해결책**: 잘못 생성된 레지스트리 값을 깨끗이 지우고, 순정 새로 만들기 CLSID인 **`{D969A300-E7FF-11d0-A93B-00A0C90F2719}`**를 정확히 주입하여 복구 완료.

#### 2. 폴더 분류 정렬 강제 우회 실패 (공백 및 느낌표 오작동)
- **증상/원인**:
  - **공백**: 탐색기 정렬 함수가 정렬 시 문자열 앞의 공백(Space)을 필터링해 무시하여 'C'(CONFIG) 등의 영어 파일 형식 그룹 아래로 여전히 밀림.
  - **느낌표**: 한글 윈도우 정렬 방식이 `[한글] ➔ [영어] ➔ [특수문자]` 순으로 동작하여, 영문 `!File Folder`로 명칭 변경 시 최하단 영역(특수문자 구역)으로 추락하여 꼴찌로 정렬됨.
- **해결책**: 사용자의 선호 및 명시적 요청에 맞춰 접두사 `#`을 조합한 **`#Folder`**로 표시 유형 명칭을 고정하여, 다른 파일 형식보다 알파벳 우선순위를 확보하여 상위 노출 정착.

#### 3. 유니코드 한글 변환 오타 및 깨짐(뷃뷄) 현상
- **증상/원인**: 파워쉘 파일 인코딩(BOM 없음 UTF-8) 충돌로 인해 문자열이 레지스트리에 저장되면서 `??` 또는 `媛 ?대뚳`와 같이 완전히 깨져 입력됨. 수동 코드 계산 과정 중 초성 인덱싱 실수로 `"가 테더"`, `"가 해더"` 같은 기괴한 명칭으로 등록되는 휴먼 에러 발생.
- **해결책**: 문자셋 인코딩 꼬임을 원천 방지하기 위해 사용자의 의견을 수용하여, 순수 아스키 문자들로 구성된 깔끔하고 안전한 영문명 **`"#Folder"`**로 최종 적용 완료.

#### 4. explorer.exe 종료 후 윈도우 검은색 먹통 버그
- **증상/원인**: 레지스트리 즉시 적용을 위해 탐색기를 `Stop-Process` 시켰으나, OS 사양에 따라 자동으로 켜지지 않고 바탕화면과 작업 표시줄이 날아가고 완전히 까맣게 되는 상태 초래.
- **해결책**: 탐색기 프로세스 킬 스크립트에 프로세스를 백그라운드로 안전하게 가동하는 **`Start-Process explorer.exe`** 코드를 동시 가동하는 안전장치를 설치하여 즉시 자동 복구되도록 처리.
