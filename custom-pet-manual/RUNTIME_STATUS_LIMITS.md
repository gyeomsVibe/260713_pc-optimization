# Codex Pet v2 9행 모션 검수

Codex Pet v2의 표준 시트에는 **서로 다른 9개 모션 행**이 있다. 이 행들은
모두 비어 있지 않고, 각자 구분되는 동선으로 조립·검수해야 한다.

| 행 | 모션 | 동선 설계 기준 |
| --- | --- | --- |
| 0 | `idle` | 주머니에 손을 넣은 편안한 기본·대기 자세의 미세한 호흡·눈 깜빡임 |
| 1 | `running-right` | 화면 오른쪽으로 진행하는 보폭이 교차하는 달리기 |
| 2 | `running-left` | 화면 왼쪽으로 진행하는 보폭이 교차하는 달리기 |
| 3 | `waving` | 한 손을 들어 인사하고 다시 내리는 짧은 제스처 |
| 4 | `jumping` | 준비 → 상승 → 공중 정점 → 하강 → 착지의 수직 동선 |
| 5 | `failed` | 고개·어깨가 처지고, 필요하면 웅크렸다가 회복하는 실패 반응 |
| 6 | `waiting` | 주머니 손을 빼고, 사용자 입력을 기다리는 질문·기대 제스처 |
| 7 | `running` | 발을 달리지 않고 손·시선·상체로 집중 처리 중임을 보이는 작업 동선 |
| 8 | `review` | 열린 팔·주먹 환호처럼 결과 완료를 명확히 전달하는 완료 동선 |

공식 화면에는 `Running`, `Needs input`, `Ready`, `Blocked`처럼 상위 활동
분류가 보일 수 있다. 이는 9개 스프라이트 행을 4개로 제한한다는 뜻이
아니다. 상위 분류와 실제 행의 선택 시점은 앱이 결정하므로, 제작물 쪽에서는
9개 행을 모두 완성하고 서로 혼동되지 않게 만들어야 한다.

## 런타임 문제를 찾는 순서

1. 설치본이 최종 후보와 같은 SHA-256 해시인지 확인한다.
2. 시트가 1536×2288, 8열×11행, RGBA, `spriteVersionNumber: 2`인지
   검사한다.
3. 0~8행의 사용 프레임 수와 인접 프레임 변화를 검사한다. 정지 그림을
   복제한 행은 실패다.
4. **Settings > Pets > Refresh** 후 새 Pet을 다시 선택한다. 기존 ID의
   이미지가 남는 문제는 새 ID로 재등록해 분리한다.
5. 작업·입력 대기·완료·오류·드래그 등 실제 화면 상황에서 Pet을 관찰한다.
   특정 상황에서 잘못된 행이 반복 선택되면 스크린샷과 함께 해당 상황을
   기록하고, 시트 행의 동선이 그 상황에 맞는지부터 고친다.

운영 체제의 **Reduce motion(동작 줄이기)** 설정이 켜져 있으면 Pet은 정지
프레임만 표시할 수 있다. 다만 일부 행만 재생되는 경우에는 이 설정 하나로
결론내리지 않고, 설치 ID·새로고침·행별 시트 검수를 모두 거친다.
