# 나만의 커스텀 반려동물(Pet) 만들기

내 캐릭터를 Codex 안에서 움직이는 반려동물로 만들고 설치하는 초보자용 매뉴얼입니다. 이 문서는 실제 제작 과정에서 생긴 시간 지연, 옷·바지 색 불일치, 크기 흔들림, 방향 오류, 붉은·보라 테두리 잔색을 다시 겪지 않도록 정리한 최종 절차입니다.

## 1. 이 작업의 목표

반려동물은 프로필 그림 한 장이 아닙니다. 여러 동작과 시선 방향을 담은 스프라이트 시트입니다. 핵심은 그림을 많이 만드는 일이 아니라 **한 캐릭터를 모든 프레임에서 같은 모습으로 유지하고, 최종 파일에 정확히 반영하는 일**입니다.

완성본은 아래를 만족해야 합니다.

- 얼굴·머리·상의·바지·신발이 모든 동작에서 같다.
- 스프라이트 시트가 `1536×2288`, 8열×11행, RGBA, v2 형식이다.
- 프레임마다 캐릭터 크기와 발 위치가 튀지 않는다.
- 동작과 화면 기준 방향이 맞다.
- 투명 배경에 초록·보라·붉은 테두리 색이 남지 않는다.
- 설치 전 백업을 만들고, 설치 후에도 같은 파일인지 확인한다.

## 2. 시작 전에 딱 세 가지만 정한다

### 기준 이미지 묶음

정면 이미지 1장, 표정·포즈가 다른 이미지 여러 장, 전체 모습을 모아 둔 합본 이미지가 좋습니다. 시작한 뒤에는 이미지 개수·파일명·기준 캐릭터를 바꾸지 않습니다. 바꿔야 한다면 새 작업으로 취급합니다.

정면·옆면·후면을 보여 주는 5컷 턴어라운드 자료는 **방향을 이해하는 방법**으로만 사용합니다. 그 안의 다른 사람·다른 캐릭터의 얼굴, 머리, 옷은 내 펫에 섞지 않습니다.

### 캐릭터 카드

그림을 만들기 전에 아래 한 장을 적습니다. 제작 중 판단이 흔들릴 때는 이 카드가 기준입니다.

| 항목 | 적을 내용 예시 |
| --- | --- |
| 이름 | 화면에 표시할 펫 이름 |
| 스타일 | 점토, 3D 토이, 스티커 등 |
| 얼굴·머리 | 눈 색, 머리 모양, 전체 분위기 |
| 상의 | 색과 구조를 한 문장으로 고정 |
| 바지·신발 | 색과 형태를 고정 |
| 금지 요소 | 후드, 지퍼, 재킷, 글자, 새 소품 등 |
| 배경 | 깨끗한 한 가지 크로마키 색 |

예: “검은 소매와 밝은 회색 몸판이 연결된 긴소매 티셔츠”처럼 적습니다. “검은 옷”이라고만 적으면 후드나 지퍼 아우터로 바뀌기 쉽습니다.

### 완료 기준

처음부터 아래 기준을 완료 기준으로 둡니다.

- 캐릭터 정체성 통일
- 동작·방향의 의미 정확성
- 크기·발 위치 안정성
- 투명 배경과 색상 잔여 제거
- 설치 전후 동일 파일 확인

## 3. 가장 빠른 제작 순서

### 1단계: 전체 일을 13개로 나눈다

일반적인 Codex v2 펫은 다음으로 구성됩니다.

1. 기준 전신 캐릭터 1개
2. 기본 동작 9개: 가만히 있기, 오른쪽 달리기, 왼쪽 달리기, 손 흔들기, 점프, 실패, 기다리기, 작업 중, 검토 중
3. 기본 방향 4컷 1개: 위·화면 오른쪽·아래·화면 왼쪽
4. 시선 방향 8컷 묶음 2개: 합쳐서 16방향

많아 보여도 전체를 반복 생성하지 않습니다. 문제가 생기면 **틀린 동작 행 전체만** 고칩니다. 한 칸만 다른 생성 결과로 끼워 넣으면 캐릭터가 갑자기 다른 사람처럼 보일 수 있습니다.

### 2단계: 생성 전에 조립·검사 환경을 확인한다

그림을 만든 후 마지막에 조립 도구가 없다는 사실을 알면 시간이 크게 늘어납니다. 시작 전에 Codex의 `hatch-pet` 흐름이 실행되는지 확인합니다. 이 흐름은 칸 자르기, 크기 정렬, 8×11 조립, 접촉 시트·움직임 미리보기, 규격 검사를 담당합니다.

처음 만드는 경우에는 이 공식 흐름 하나를 따릅니다. 임의의 보조 스크립트를 여러 개 섞어 쓰지 않습니다.

### 3단계: 기본 동작 9개를 한 행씩 확인한다

각 행을 만든 직후 아래만 확인합니다.

- 기준 캐릭터와 같은 얼굴·머리·옷·바지·신발인가?
- 몸이나 손발이 칸 밖으로 잘리지 않았는가?
- 프레임마다 크기와 발 위치가 크게 바뀌지 않는가?
- 같은 정지 그림의 반복이 아닌가?
- 동작의 뜻이 맞는가?

동작별 주의점:

- `running-right`와 `running-left`는 보는 사람 화면 기준으로 각각 오른쪽·왼쪽을 향해야 합니다.
- `running`은 실제 달리기가 아니라 작업·처리·집중을 나타냅니다. 속도선이나 달리는 자세를 넣지 않습니다.
- `waving`은 손·팔 자세만으로 인사합니다. 파동선·반짝이·말풍선은 넣지 않습니다.
- `jumping`은 몸 높이만 바꿉니다. 바닥 그림자·먼지·착지 효과는 넣지 않습니다.

### 4단계: 방향 16개는 화면 기준으로 만든다

방향의 왼쪽·오른쪽은 캐릭터 기준이 아니라 **보는 사람 화면 기준**입니다. 먼저 위·화면 오른쪽·아래·화면 왼쪽의 네 방향이 분명한지 확인하고, 그 사이 방향을 만듭니다.

`위 → 오른쪽 위 → 오른쪽 → 오른쪽 아래 → 아래 → 왼쪽 아래 → 왼쪽 → 왼쪽 위`

눈동자만 움직여 억지로 방향을 만드는 방식은 피합니다. 얼굴의 코·눈·머리 실루엣이 자연스럽게 방향을 보여야 합니다.

### 5단계: 최종 시트는 한 번 조립한다

프레임을 고쳤다면 최종 스프라이트 시트, 접촉 시트, GIF 미리보기도 함께 다시 만듭니다. 오래된 접촉 시트는 최신 프레임을 반영하지 않을 수 있으므로 근거로 쓰지 않습니다.

최종 시트는 `1536×2288`, 8열×11행, RGBA여야 합니다. 빈 칸은 완전히 투명해야 하고, v2 설정을 사용해야 합니다.

### 6단계: 테두리 잔색은 마지막에 한 번만 정리한다

배경을 투명하게 만들 때는 최종 조립본을 대상으로 한 번만 잔색 보정을 합니다. 여러 번 보정하면 머리카락·신발·피부 가장자리도 흐려질 수 있습니다.

초록색 배경을 썼는데 보라색 또는 붉은 테두리가 남는다면 초록색 검사만 믿지 않습니다. 실제 남은 색을 기준으로 검사한 뒤, 원본 최종 시트에서 한 번만 다시 보정합니다. 보정 후에는 다음을 확인합니다.

- 투명 윤곽이 원본과 같은가?
- 초록색과 보라·붉은색 잔색 검사가 모두 통과하는가?
- 8×11 규격과 사용 칸이 그대로인가?

### 7단계: 백업 후 설치하고 다시 확인한다

기존 `spritesheet.webp`를 날짜가 붙은 이름으로 백업합니다. 새 시트와 `pet.json`을 함께 설치합니다.

```text
<CODEX_HOME>\pets\<내-펫-아이디>\
  pet.json
  spritesheet.webp
```

`pet.json` 최소 예시:

```json
{
  "id": "my-pet",
  "displayName": "My Pet",
  "description": "My custom Codex companion.",
  "spriteVersionNumber": 2,
  "spritesheetPath": "spritesheet.webp"
}
```

복사 뒤에는 설치 폴더의 파일을 다시 검사하고, 후보 파일과 설치본의 SHA-256 값이 같은지 확인합니다. Codex에서 다른 펫을 한 번 선택했다가 내 펫을 다시 선택하거나 앱을 재시작해 새 그림을 불러옵니다.

## 4. 5분 설치 전 체크리스트

- [ ] 기준 이미지와 캐릭터 카드를 확정했다.
- [ ] 개인 사진과 생성 원본은 Git 저장소 밖에 둔다.
- [ ] 모든 행에서 옷·바지·신발 색이 동일하다.
- [ ] 잘린 몸, 겹친 포즈, 빈 프레임이 없다.
- [ ] 기본 9개 동작이 실제로 움직인다.
- [ ] 4개 기본 방향과 16개 방향이 화면 기준으로 맞다.
- [ ] 최종 시트가 1536×2288, 8×11, RGBA, v2이다.
- [ ] 초록·보라·붉은 잔색이 없다.
- [ ] 기존 설치본을 백업했다.
- [ ] 설치 전 파일과 설치 후 파일의 해시가 같다.
- [ ] Codex 안에서 실제 표시를 확인했다.

## 5. 실제 실패 사례와 해결법

| 증상 | 원인 | 다음에는 이렇게 한다 |
| --- | --- | --- |
| 작업 시간이 16시간 이상 길어짐 | 기준 이미지·파일명이 중간에 여러 번 바뀜 | 시작 전에 기준 묶음과 캐릭터 카드를 고정한다. |
| 한 동작만 다른 캐릭터처럼 보임 | 다른 생성 결과를 한 칸만 섞음 | 문제 행 전체를 같은 소스에서 다시 만든다. |
| 바지가 파랗거나 옷이 후드처럼 변함 | 색·옷 구조가 고정 조건으로 적히지 않음 | 의상 구조와 금지 요소를 캐릭터 카드에 명시한다. |
| 캐릭터가 프레임마다 커졌다 작아짐 | 프레임마다 따로 확대·축소함 | 한 행에는 하나의 공통 비율만 적용한다. |
| 프레임은 고쳤는데 결과가 그대로임 | 오래된 스프라이트·GIF를 다시 봄 | 프레임 변경 뒤 최종 시트와 미리보기를 함께 갱신한다. |
| 구조 검사는 통과했지만 애니메이션이 멈춰 보임 | 빈 칸만 검사함 | 인접 프레임 변화와 미리보기를 모두 확인한다. |
| 왼쪽·오른쪽이 반대로 보임 | 캐릭터 기준과 화면 기준을 섞음 | 모든 방향 검사에 “화면 기준”을 적는다. |
| 붉은·보라 테두리가 남음 | 한 가지 배경색만 검사하거나 여러 번 보정함 | 실제 잔색을 검사하고 원본 최종 시트에서 한 번만 보정한다. |
| 설치 후에도 예전 그림이 보임 | 앱이 기존 이미지를 표시함 | 다른 펫을 선택 후 다시 선택하거나 앱을 재시작한다. |
| 작은 확인마다 작업이 느려짐 | 파일·검사를 하나씩 왕복 실행함 | 읽기·검증은 묶어서 하고, 실패한 행만 고친다. |

## 6. 문제가 생겼을 때 빠른 판단법

1. 먼저 “그림 자체가 틀렸는지”와 “조립·설치가 틀렸는지”를 구분합니다.
2. 그림 자체가 맞다면 새로 생성하지 말고, 현재 프레임으로 시트·미리보기·검사 결과를 다시 만듭니다.
3. 크기·빈 칸·투명도처럼 계산으로 고칠 수 있는 문제는 먼저 자동 보정·검사로 해결합니다.
4. 얼굴·옷·방향처럼 그림이 잘못된 문제만 해당 행을 다시 만듭니다.
5. 같은 실패가 두 번 반복되면 문구만 바꾸며 다시 시도하지 말고, 포즈·소품·기준 이미지·추출 방법 중 하나를 단순하게 바꿉니다.
6. 같은 원인으로 세 번 실패하면 로그·원인·우회 방법을 기록하고 전략을 바꿉니다.

## 7. GitHub에 올려도 되는 것

올려도 되는 것:

- 이 매뉴얼과 익명화한 체크리스트
- 일반적인 실패 사례와 해결 방법
- 개인 정보가 없는 `pet.json` 예시

올리면 안 되는 것:

- 개인 사진과 개인 캐릭터 원본
- 생성 과정의 원본 이미지와 임시 시트
- 사용자 이름·절대 경로가 담긴 설치 기록
- 실제 설치 폴더의 펫 파일
- API 키, 토큰, 비밀 설정
