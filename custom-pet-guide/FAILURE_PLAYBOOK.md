# 실패 사례와 재발 방지책

| 실패 | 원인 | 재발 방지책 |
|---|---|---|
| 기준 이미지가 5→6→7장으로 반복 변경 | 입력 정본과 파일명이 고정되지 않음 | 시작 전에 파일명·개수·해시를 동결한다. |
| 검은 상의를 후드·지퍼 아우터로 생성 | 의상 해석이 자연어에만 의존 | 검은 소매와 밝은 회색 몸판, 후드·지퍼 금지를 계약으로 적는다. |
| 전체 생성이 반복되어 시간이 폭증 | 한 프레임 오류도 전체 아틀라스로 재생성 | 문제가 있는 행만 분리 교체한다. |
| running-left만 다른 캐릭터처럼 보임 | 서로 다른 생성 결과를 셀 단위로 혼합 | 같은 소스 스트립의 완전한 행으로 교체한다. |
| 점프 프레임의 바지가 파란색 | 색상 잠금이 동작 프롬프트보다 약함 | 검은 바지를 불변 조건으로 두고 행 전체를 검사한다. |
| 행별 높이가 122~198px로 제각각 | 프레임마다 개별 확대·축소 | 한 행에는 하나의 공통 배율만 적용한다. |
| 방향 검수가 좌우 반대로 판정 | 캐릭터 기준과 화면 기준을 혼용 | 화면 왼쪽·오른쪽 좌표와 발 진행 방향을 함께 확인한다. |
| 정지 프레임인데 구조 검증 통과 | 셀 존재 여부만 검사 | 인접 프레임 픽셀 변화율을 별도 측정한다. |
| 크로마키 색이 머리·신발 가장자리에 남음 | 합성 중 디스필을 여러 번 수행 | 최종 합성 후 한 번만 디스필하고 잔여 픽셀을 측정한다. |
| `r0c6` 중립 프레임이 삭제됨 | 일반 사용 프레임 수만 보고 미사용 칸 정리 | v2 특수 중립 프레임을 예외로 보존한다. |
| JSON 상태 파일이 잘못 덮어써짐 | 보고서와 정본 메타데이터 경로 혼용 | 출력 보고서와 `pet.json` 경로를 분리한다. |
| 승인·도구 호출이 과도하게 반복됨 | 작은 작업마다 왕복 승인과 재검사를 수행 | 입력 동결 후 읽기·검증은 묶고, 외부 부작용만 별도 승인한다. |
| 같은 도구 오류를 무한 재시도 | 실패 횟수 제한과 우회 조건이 없음 | 동일 실패 3회 후 로그·원인·우회책 순으로 전환한다. |

## 핵심 원칙

캐릭터 정체성, 동작, 크기, 배경 투명성은 서로 다른 품질 축입니다. 구조 검증 통과를 시각 품질 통과로 간주하지 말고, 자동 수치 검증과 접촉 시트 검수를 모두 수행합니다.
